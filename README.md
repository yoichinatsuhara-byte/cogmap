# CogMap — 高次脳機能 統合プロファイル（MVP）

高次脳機能評価（CAT・BADS・TMT-J・SLTA・WAIS など）の結果を入力すると、
**「検査項目 → 認知領域」のマッピング**に基づく統合プロファイルを可視化する
院内ローカルWebアプリです。OT/ST 間で「どの検査がどの機能を反映しているか」
「まだ誰も見ていない機能（未評価領域）はどこか」を共有することを目的とします。

- 完全ローカル（外部通信ゼロ）・ビルド不要・依存ゼロ（Vanilla JS）
- データはブラウザの `localStorage`（キー `cogmap.v1`）に保存
- 共有は **同一端末での閲覧** と **印刷ビュー（A4印刷）** で行います
  （JSONエクスポート/インポートはv2で廃止しました。他端末での共有が必要になった場合の復活候補です）

## 安全原則（重要）

1. **非PII**: 患者氏名・患者ID・生年月日は入力しません。ケースは匿名ラベル（例「A-01」）のみ。
   氏名等の個人情報は絶対に入力しないでください。
2. **診断しない**: 本ツールは評価者が入力した判定の整理・可視化であり、
   自動診断・重症度判定は行いません。
3. **カットオフの自動判定なし**: 判定（正常域／境界／低下／顕著な低下）は、
   各検査マニュアルの基準に基づき **評価者自身が選択** します。
4. **完全ローカル**: CDN・Webフォント・外部API・クラウド同期は一切使いません。
   配信は `127.0.0.1`（ローカルホスト）のみです。

## 起動方法

`起動.command` をダブルクリックします。
`127.0.0.1:8418` でローカルサーバーが立ち上がり、既定ブラウザで開きます。

- Python 3 が必要です（macOS 標準の `python3`）。
- ポート 8418 で CogMap がすでに起動済みの場合は、それを再利用します。
  無関係な別アプリがポート 8418 を使用している場合は、再利用も起動もせずエラーで案内します。
- コマンドラインからは、このフォルダで `python3 -m http.server 8418 --bind 127.0.0.1`
  を実行し、`http://127.0.0.1:8418/` を開いても同じです。

> 注: `file://` で直接開くと一部ブラウザで動作しないため、必ずローカルサーバー経由で開いてください。

## 使い方

1. **ケース** — 匿名ラベル（例「A-01」）とメモでケースを作成し、選択します。
   個人情報は入力しないでください。
2. **評価入力** — 検査を選び、実施日・職種（OT/ST）・実施者を入力。
   下位項目ごとに判定（正常域／境界／低下／顕著な低下／未実施）を選び、
   必要なら生値・コメントを添えて保存します。入力済みセッションは編集・削除できます。
   出典が確認できているカットオフ・基準がある下位項目には「目安」（出典つき）を表示し、
   生値を入れると該当する判定ボタンを点線＋「目安」タグでハイライト提案します。
   **これはあくまで参考であり、判定ボタンは自動選択されません。最終判定は必ず評価者（セラピスト）が行ってください。**
   目安が無い項目は「基準はマニュアル参照」と表示されます。
   各下位項目の「基準を見る」リンクから、目安の詳細・出典・施設基準をその場（ポップオーバー）で確認・編集できます。
   施設基準に数値バンドを登録した項目では、生値入力時に**施設基準の提案が内蔵の目安より優先**され、
   実線＋「施設基準」タグで（内蔵の目安は点線＋「目安」タグで）区別して表示されます。
   収載検査は `data/assessments.js` の一覧に準じます（HDS-R・語流暢性検査は現時点では収載外です）。
3. **プロファイル** — 大分類レーダー（6軸）＋領域別バー（22領域・データ定義に準拠）。
   各領域をクリックすると根拠（どの検査のどの項目・職種・日付・判定）が展開します。
   「22領域中◯評価済み」のカバレッジと未評価領域、OT/ST実施内訳も表示します。
4. **マトリクス** — 認知領域 × 実施済み検査（検査×職種の列。例: CAT[OT]とCAT[ST]は別列）の
   反映マップ。同一検査でもOT実施とST実施を両方確認できます。セルの色＝最新判定、
   大きさ＝関与の強さ（重み 1.0/0.6/0.3）。BADS総プロフィール得点・WAIS FSIQ・WAB AQ など
   二重カウント防止のため参考扱い（scoring: reference）とした指標は列に出しません。
5. **経過** — 評価日が2日以上あるとき、大分類スコアの推移を折れ線で表示します。
6. **ADLの視点** — 「低下」以上の領域について、生活場面で起きやすいこと・
   観察のポイント・工夫の候補を提示します（あくまで候補で、断定ではありません）。
7. **基準** — 収載する全検査×下位項目の判定目安（テキスト・数値バンド・出典・信頼度・
   「要人間確認」フラグ）を検査ごとのアコーディオンで一覧参照できます。
   カットオフや年齢基準をマニュアルからいちいち引かなくても、この画面や評価入力の
   「基準を見る」からクリックひとつで確認できます。あわせて各項目に **施設基準（マイ基準）** を
   登録できます（下記）。
8. **印刷** — プロファイル＋簡約マトリクスを A4縦1枚のサマリーとして印刷します。

### 判定のブレ対策（v4）

検査ごと・評価者ごとに判定がぶれるのを抑えるため、次の4つを備えています。

- **判定アンカー（運用定義）**: 評価入力画面の上部に、正常域／境界／低下／顕著な低下の
  「運用上の意味づけ」を常時表示します（折りたたみ可）。各判定ボタンにも同じ定義を
  ツールチップで表示します。判定は評価者が確定するもので、結論を強制するものではありません。
- **判定根拠チップ**: 各下位項目に、判定の根拠（数値基準／質的誤反応／ADL・生活場面／
  他職種情報／検査条件（疲労・失語等））を任意・複数で付けられます。セッション一覧と
  プロファイルの根拠展開に表示され、「なぜその判定にしたか」を後から追えます。必須ではありません。
- **判定不一致の検出**: 同一領域で「正常域」と「低下以上」が混在する、または OT と ST の
  判定が2段階以上離れている領域を「不一致」バッジで示し、根拠展開にどの検査・職種が
  割れているかの内訳を表示します。
- **臨床サマリー5行**: プロファイル最上部に、①主要な低下（最悪判定を併記）②安全・ADLに
  直結する領域の低下／未評価 ③未評価の重要領域と推奨検査（逆引き）④判定不一致 ⑤次の観察、
  を自動整理して表示します（診断・重症度の自動判定ではありません）。
- 領域別バーは数値でなく **言葉（低下・根拠n件）を主表示**し、平均で重症度が薄まる場合に
  そなえて「最悪: 顕著な低下」を併記します。総合指標（例: 総プロフィール点）は
  二重カウントを避けるため計算から除外し、根拠には「参考」として表示します。

### 施設基準（マイ基準）について

年齢別基準など、マニュアル参照が必要な値を施設で一度だけ登録し、以後はクリック参照・
ハイライト提案に使える機能です。登録した施設基準は `localStorage`（キー `cogmap.norms.v1`）に
全ケース共通で保存されます。次の3点を必ず守ってください。

- **値はユーザー（施設）が入力します。** メモ（1行）と任意の数値バンド（判定×生値範囲）を
  評価者自身が入力します。
- **アプリは基準の既定値を持ちません。** 施設基準の初期値は空です。本アプリが年齢別正規表や
  カットオフを埋め込んで提供することはありません（捏造防止の原則）。
- **出典の管理は施設側の責任です。** 入力した値がどのマニュアル・ページに基づくかは、
  メモ欄に記録するなどして施設側で管理してください。

## OT ⇔ ST 共有手順

CogMap はアカウントやクラウド同期を持たず、データはブラウザの `localStorage` に
その端末だけで保存されます。OT/ST 間の共有は **同一端末での閲覧** と **印刷（A4）** で行います。

1. 同一端末で開いて共有する: 院内の共有PC・共有ブラウザで CogMap を開けば、
   OT・ST どちらもケース・入力済みセッション・プロファイルをそのまま閲覧できます。
2. 紙で共有する: **印刷** タブで A4縦1枚のサマリー（プロファイル＋簡約マトリクス）を
   印刷し、カルテや申し送りに貼って共有します。

> 別端末（自分のPC等）へのデータ持ち出しが必要になった場合は、JSONエクスポート/インポート機能の
> 復活を検討します（v2で廃止・`docs/SPEC.md` §14 参照）。

## 更新履歴

- **v5**: 保存に失敗したとき（ブラウザの保存領域の問題など）、無言で消えず画面にエラーを表示するようになりました。
- **v5**: マトリクスの列が「検査×職種」単位になり、同一検査のOT実施・ST実施を両方確認できるようになりました。
- **v5**: 評価入力の実施日に未来日を入力できなくなりました。
- **v5**: 印刷の大分類マトリクスが複数のブロックに分割され、A4で表が途中で切れなくなりました。
- **v5**: BADS総プロフィール得点・WAIS FSIQ・WAB AQ など参考扱いの指標を、マトリクス・印刷からも除外しました（プロファイル計算からは従来どおり除外）。
- **v5**: `起動.command` がポート8418の応答をCogMap本体かどうか確認するようになり、無関係な別アプリがポートを使用中の場合に誤って再利用・起動しなくなりました。

## ファイル構成

```
index.html            画面（SPA・上部タブ）
css/style.css         スタイル（クリニカル・スイス／印刷CSS）
js/util.js            共通ユーティリティ（esc・判定ラベル・閾値・表示桁の一本化）
js/store.js           localStorage 永続化
js/profile.js         プロファイル計算（純関数）
js/app.js             画面制御・ルーティング
data/domains.js       認知領域マスタ（COG_DOMAINS / COG_GROUPS）
data/assessments.js   検査マスタ（COG_ASSESSMENTS）
data/adl-map.js       ADLマップ（COG_ADL_MAP）
js/viz.js             可視化（レーダー・バー・マトリクス・経過）
起動.command          ローカル配信ランチャー（127.0.0.1:8418）
docs/SPEC.md          設計書（正典）
```

本ツールは評価結果の整理・共有を支援するものであり、診断や重症度の自動判定は行いません。
判定は各検査マニュアルの基準に基づき評価者が入力してください。
